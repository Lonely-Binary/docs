<Module n="02" title="Code example" est="~ 90 sec · 3 frameworks">

Same sketch, three ways. Pick the framework you're already in. We've tested all three on the LB-32 board — none need extra libraries.

<CodeTabs>

<CodeBlock id="arduino" tab="sketch.ino" ext="· Arduino" file="sketch.ino · 24 lines" lang="cpp" downloadName="sketch.ino" downloadLabel="Download .ino">

```cpp
// sketch.ino  — pair, then toggle the LED on incoming bytes
#include <BluetoothSerial.h>

BluetoothSerial BT;
const int LED = 2;

void setup() {
  pinMode(LED, OUTPUT);
  Serial.begin(115200);
  BT.begin("LB-ESP32");   // device name your phone sees
  Serial.println("Ready. Pair with LB-ESP32 from your phone.");
}

void loop() {
  if (BT.available()) {
    char c = BT.read();
    digitalWrite(LED, !digitalRead(LED));
    BT.printf("got %c (0x%02X)\n", c, c);
    Serial.printf("phone → board: %c\n", c);
  }
}
```

</CodeBlock>

<CodeBlock id="pio" tab="main.cpp" ext="· PlatformIO" file="main.cpp · PlatformIO" lang="cpp" downloadName="main.cpp" downloadLabel="Download project">

```cpp
// platformio.ini → board = esp32dev, framework = arduino
#include <Arduino.h>
#include <BluetoothSerial.h>

BluetoothSerial BT;
constexpr int LED = 2;

void setup() {
  pinMode(LED, OUTPUT);
  Serial.begin(115200);
  BT.begin("LB-ESP32");
}

void loop() {
  if (BT.available()) {
    char c = BT.read();
    digitalWrite(LED, !digitalRead(LED));
    BT.printf("got %c\n", c);
  }
}
```

</CodeBlock>

<CodeBlock id="mpy" tab="main.py" ext="· MicroPython" file="main.py · MicroPython · BLE" lang="python" downloadName="main.py" downloadLabel="Download .py">

```python
# main.py — BLE peripheral version (Classic BT isn't on MicroPython)
import bluetooth, machine, time
from machine import Pin

led = Pin(2, Pin.OUT)
ble = bluetooth.BLE()
ble.active(True)

def on_event(event, data):
    if event == 3:            # _IRQ_GATTS_WRITE
        led.value(not led.value())

ble.irq(on_event)
print("Advertising as LB-ESP32 · open nRF Connect to talk to it")
```

</CodeBlock>

</CodeTabs>

The whole sketch is **24 lines**. Three of them name your device, define the LED pin, and start Bluetooth. The loop just toggles the LED on every incoming byte.

<Module n="03" title="Bluetooth init & pairing" est="~ 90 sec · 4-stage flow">

Pairing is four phases. The active phase advances every few seconds — or tap a card to focus it.

<PairingFlow autoAdvance interval={3500}>

<Phase n="01" title="BT.begin()" anim="advertise">

ESP32 turns on the BT radio and starts advertising as `LB-ESP32`.

</Phase>

<Phase n="02" title="Phone scans" anim="scan">

Your phone listens for advertisements. Yours pops up under "Available devices".

</Phase>

<Phase n="03" title="Pair & bond" anim="bond">

Tap the device. Confirm the pairing prompt. A bond key is stored on both sides.

</Phase>

<Phase n="04" title="Channel open" anim="duplex">

Bytes can flow both ways. `BT.available()` returns true on the ESP32.

</Phase>

<Module n="05" title="Common issues" est="~ 45 sec · 5 fixes">

The five things that trip up 90% of beginners. Each one is a one-line fix.

<Troubleshoot>

<Fix title="LB-ESP32 doesn't show up in my phone's BT list">

Your sketch needs `BT.begin("LB-ESP32")` in `setup()`. If it's there, the radio may be busy: power-cycle the board and turn BT off/on on your phone. iOS sometimes caches old devices — "Forget Device" and re-scan.

</Fix>

<Fix title="Pairs, but nothing happens when I send data">

Most phone BT chat apps default to BLE — but Classic BT and BLE are different protocols. Either install **Serial Bluetooth Terminal** (Android) which speaks Classic, or switch your sketch to `BLEDevice::createServer()` (BLE example in the docs).

</Fix>

<Fix title={<>Compile fails: <code>BluetoothSerial.h: No such file</code></>}>

You picked an ESP32 variant that doesn't have Classic BT (e.g. ESP32-S2). Switch the board to `ESP32 Dev Module` in Tools, or use the BLE version of the sketch — S2/S3 boards only support BLE.

</Fix>

<Fix title="Sketch flashes but board keeps rebooting">

Partition table is too small. Tools → Partition Scheme → **Huge APP (3MB No OTA / 1MB SPIFFS)**. Classic BT brings in a fat stack and the default partition won't fit.

</Fix>

<Fix title="Pairing succeeds but disconnects after a few seconds">

Likely under-powered USB. Cheap hubs can't deliver 500 mA during BT bursts. Try a different cable, plug straight into the laptop, or power the board from an external 5V supply.

</Fix>

</Troubleshoot>
